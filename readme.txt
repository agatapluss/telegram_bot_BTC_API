

Так как у нас ожидается небольшой функционал, а именно создание кошельков в телеграм боте и отправка транзакций, то думаю нам нужны будут такие сущности:

User - сущность пользователя, она должна хранить в себе такие данные: Внутренний ID в нашей бд,  id в телеграме, ник юзера, дату создания аккаунта в боте, кошелек, историю транзакций 

Wallet - сущность кошелька, она должна хранить в себе такие данные: Внутренний ID в нашей бд, кто владелец кошелька, баланс, приватный ключ, адрес кошелька, а также все транзакции совершенные с этого кошелька. 

Transaction - сущность транзакции, она должна хранить в себе такие данные: Внутренний ID в нашей бд, юзера-отправителя, кошелек с которого была отправлена транзакция,кошелек куда отправлены биткоины, отправленная сумма, комиссия за транзакцию, дату создания транзакции, хеш транзакции (по которому её можно отследить в блокчейне)

У нас выходит всего 3 сущности, но на их примере вы сможете увидеть как строить базы данных с помощью Пони.



## Функционал для работы бота:

1. Создание транзакций
2. Отображение адреса кошелька для получения транзакций
3. Проверка баланса
4. Импорт приватного ключа пользователем
5. Получение истории транзакций
6. Удаление юзера из бота
